Critical constraint — read before building anything
YouTube completely blocked Watch Later from the official Data API v3 in 2016. playlistItems.list(playlistId:"WL") returns empty always. Every feature in this spec is built via DOM manipulation inside a content script — not the official API. This is how every successful Watch Later extension works today.
DOM Content script reads page HTML directly — most stable method
Innertube YouTube's internal API — unofficial but widely used

06
Technical Architecture
Reference
📄
content_script.js
  • Runs inside youtube.com pages
  • DOM reading, MutationObserver, event listeners
  • Trusted Types policy for all DOM writes
  • Sends messages to background service worker
  • Injected on: /playlist?list=WL, /watch, /, /results
⚙️
service_worker.js
  • Manifest V3 background script
  • Handles chrome.storage reads/writes
  • Manages auth state for Pro features
  • Calls LemonSqueezy / Stripe webhook verification
  • No persistent state — event-driven only
🗄️
Storage Strategy
  • chrome.storage.sync → user prefs, sort order, priority queue (100KB limit)
  • chrome.storage.local → firstSeen dates, stale flags (no limit)
  • IndexedDB → overflow WL videos beyond 5k, full video index
  • No server-side storage in MVP
🔌
Popup UI
  • Built as popup.html — React or vanilla JS
  • Shows WL health dashboard stats
  • Houses session builder input
  • Settings panel (thresholds, sort prefs)
  • Pro upgrade CTA with Stripe link
Trusted Types — build this right from day one
YouTube enforces Trusted Types since July 2024. Any element.innerHTML = rawString will be blocked and silently fail. Always use: const p = trustedTypes.createPolicy('wl-ext', { createHTML: s => s }) then element.innerHTML = p.createHTML(yourMarkup). Non-compliant extensions break on YouTube with no error shown to users.

Risk Register
Know Before You Build
Risk
Severity
Likelihood
Mitigation
YouTube DOM structure changes
YouTube updates HTML layout 2–4× per year. Selectors break silently.
High impact
Certain
Use multiple fallback selectors. Set up automated Playwright test on WL page. Budget for 1–2 emergency patches/year. Monitor YouTube's design Twitter account.
Trusted Types enforcement tightens
Google may tighten Trusted Types policy further, requiring additional compliance work.
Low impact
Low
Build Trusted Types compliance from day one — not as an afterthought. Follow Chrome Extensions team blog for policy changes.
Rate limiting on bulk delete
Deleting 500+ videos too fast may trigger CAPTCHA or temporary account throttle.
Medium impact
Medium
Hard-cap at 2 deletions/second. Show progress bar with time estimate. Add pause/resume. Alert user if CAPTCHA detected via DOM watch.
YouTube ToS grey area
ToS prohibits automated interactions. DOM manipulation by extensions is a grey zone.
Low impact
Very Low
100+ extensions do exactly this (SponsorBlock, uBlock, PocketTube). Google's Chrome team works with extension devs. Never automate account-level actions at scale from a server. All actions must be user-initiated.
